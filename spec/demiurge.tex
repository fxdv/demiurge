\begin{abstract}
One assertion drives every decision below: \textbf{an inference request is not a
packet, it is a lease on stateful accelerator memory whose duration is unknown at
admission.} Demiurge exploits three facts that round-robin and least-connections
ignore: the most valuable backend state (the KV cache) is request-correlated, the
cost of a request depends on the target's live batch and KV footprint, and
occupancy is a random variable. The balancer schedules prefill and decode as
independent phases across two pools, treating the KV cache as the explicit
hand-off artifact between them.
\end{abstract}

\section{Architecture}
Three planes with three consistency models and three blast radii: a microsecond
data plane (eBPF/XDP admission plus a Rust \texttt{io\_uring} L7 forwarder serving
an RCU snapshot), a millisecond control plane (cost evaluation, length
prediction, pool-ratio autoscaling) under a CP consensus, and an
eventually-consistent (AP) state plane carrying the KV warmth map and live
occupancy. A wrong state-plane guess costs a cache miss, never a correctness
violation. Authorization for cache sharing is the one fact kept strongly
consistent.

\section{Cost Function}\label{sec:cost}
The routing cost is a product of a strictly-positive time core and dimensionless
factors:
\[
  C(r,q) \;=\; T_{\text{core}}(r,q)\;\cdot\;\prod_m B_m\;\cdot\;\prod_c D_c\;\cdot\;\delta,
  \qquad B_m \ge 1,\; D_c \in (0,1],\; \delta \in [1-\alpha,\,1+\alpha].
\]
Because $T_{\text{core}}>0$ and every factor is strictly positive, $C(r,q)>0$ by
construction \req{DEMI-COST-POS}. Rewards enter only as multiplicative discounts
$D_c\in(0,1]$, never as subtractions, so no reward can drive cost negative and
invert the corrector. The learned corrector $\delta$ is clamped to
$[1-\alpha,1+\alpha]$ \req{DEMI-CORR-CLAMP}, so it can adjust the analytic cost by
at most a bounded multiplicative envelope.

\section{Cross-Tenant Cache Sharing}\label{sec:sharing}
Prefix-cache reuse across tenants is opt-in via a Shared-Prefix Group with a
content-verified template. The data plane derives cache-domain keys from a
tenant-salted fingerprint, and a non-member can never resolve to a shared
cache-domain key \req{DEMI-S1-DOMAIN}; membership is checked on the strongly
consistent authorization path.

\section{Routing and Migration}\label{sec:routing}
\texttt{Route} admits the request, dispatches prefill asynchronously, and returns
immediately without blocking the forwarder \req{ALG-ROUTE}; decode placement is a
continuation evaluated on fresh telemetry once prefill completes.

\begin{algorithm}[h]
\caption{Route (non-blocking dispatch)}
\begin{algorithmic}[1]
\State \textbf{on} request $r$:
\State admit $r$ under tenant token bucket \textbf{or} shed at XDP
\State $p \gets \textsc{SelectPrefillTarget}(r)$
\State \textbf{dispatch} prefill of $r$ to $p$ \textbf{asynchronously}
\State \textbf{return} \Comment{decode placed later in \textsc{OnPrefillComplete}}
\end{algorithmic}
\end{algorithm}

Live migration of a decode is chunked and abortable: the cutover commits only if
the estimated stall stays within a fraction $\varepsilon$ of the inter-token
latency budget \req{DEMI-MIG-SUBITL}; otherwise it aborts and leaves the original
placement untouched.
